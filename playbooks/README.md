# `playbooks/` — Living Operational Playbooks

Operational assets for running the business — **Design Partner validation first**, more to follow.

**These are living documents.** Unlike the **Product Constitution** (`CLAUDE.md`, `docs/STRATEGY.md`,
`docs/VISION.md`, …), which is *stable* and changes only through the deliberate Thesis-Evolution
protocol, playbooks are **expected to evolve continuously through evidence and field experience**.
Update them freely as sessions teach us what works.

| Layer | Where | Changes |
|---|---|---|
| **Product Constitution** (strategy, Thesis, invariants) | `CLAUDE.md`, `docs/` | Rarely — via evidence-backed Thesis Evolution only |
| **Operational Playbooks** (how we run validation/sales/ops) | `playbooks/` | Continuously — as the field teaches us |

**Rule of thumb:** if a change alters *what NH is or how a recovered dollar is counted*, it belongs in
the Constitution. If it alters *how we run a customer session or capture evidence*, it belongs here.
