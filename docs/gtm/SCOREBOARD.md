# Validation Scoreboard

**The product is frozen. This is the only board that moves.**

Update this — and nothing else — every week. Not screens added, not fields added, not
docs written, not commits. Those are *motion*, not progress. The bottleneck is no longer
the product; it is whether the market wants it.

---

## The build trigger — the most important rule in the project

> **No sentence authorizes building product. Only a _defined pilot_ does.**

*"Show me that on our data"* is the most exciting thing you'll hear — and it still does
**not** mean "build." It opens the **data-access** step. Building waits for a pilot.

| They say… | You do… |
|---|---|
| "Interesting." · "Nice idea." · "Cool." · "Makes sense." | **Nothing.** Keep talking. |
| "How do you prove that?" | Open the CFO Proof View → the case ledger. Don't build. |
| "Can you run this on our data?" | **Ask for the data.** Don't build yet — open access. |
| *(they actually send the CSV)* | Run the Leak Assessment (existing method). Still no product build. |
| *(a pilot is defined)* | **Now you build** — and only what the pilot needs. |

Before any product is built, five checks must clear:
1. What data do they actually have? 2. Can a Leak Assessment be produced from it?
3. Is the problem real? 4. Will they invest time? 5. Is a pilot defined?

Curiosity is not demand. A request to touch their data is interest, not commitment.

> and the gap between *Asked to see it on their data* and *Shared real data* is where
> most validation quietly dies.

```

```

**The one number to watch daily** is not Paying Customers — it is **Shared real data
(CSV in hand)**. The day it goes `0 → 1`, the conversation stops being about an idea and
starts being about a business.

## The order we hold — zealously

```
Conversation → Data Access → Leak Assessment → Pilot → Build Again
```

Never skip left to right. The Leak Assessment uses the **existing** method
([`ROI_CALCULATOR.md`](ROI_CALCULATOR.md), one CSV) — it is *not* a product build. The
biggest risk now is psychological, not technical: the product is starting to *feel* real,
and a real-feeling product is seductive to keep polishing instead of testing whether
anyone wants it. **Polishing is hiding. This scoreboard is the cure.**
